## Big Picture
You have:
- `frontend` (JavaFX UI)
- `backend` (server + business logic + DB access)
- database SQL files, docs, configs

The backend is separated into clear layers so each part has one job.

---

## Why each folder exists

- `model`
  - Core domain objects (`User`, `Student`, `Attendance`, etc.)
  - Represents real business entities

- `dto` (Data Transfer Object)
  - Objects used to transfer data between client and server
  - Not full DB/domain objects, only needed fields

- `dto/requestDto`
  - Input payloads from frontend to backend
  - Example: login request contains username/password

- `dto/responseDto`
  - Output payloads from backend to frontend
  - Example: success flag, message, data

- `dao` (Data Access Object)
  - Direct DB code (SQL queries, inserts, updates)
  - Keeps SQL isolated from business logic

- `service`
  - Business rules
  - Calls DAO, applies validations, calculations, workflows

- `command`
  - Your app’s request handling pattern
  - Maps incoming command names (`Login`, `GetBatchAttendance`) to handlers
  - Similar to “controller layer” in REST apps

- `server`
  - Socket client handling / server startup

- `utility`
  - Shared helpers (JWT, DB pool, token blacklist, etc.)

---

## Why Request/Response DTO (instead of sending model directly)

- Security: hide sensitive fields (password hash, internal IDs)
- Stability: frontend contract stays stable even if DB model changes
- Validation: request DTOs can enforce required input format
- Cleaner API: response can include `success`, `message`, and `data` consistently

---

## Why DAO + Service split

- DAO = **how to get/save data**
- Service = **what business decision to make**

Example:
- DAO: `getStudentAttendanceByBatch(batch)`
- Service: “attendance < 80% and no medical => not eligible”

This separation makes testing and debugging much easier.

---

## Simple flow in your app

Frontend button click  
-> sends request command  
-> backend `Command` handler  
-> `Service` method  
-> `DAO` query DB  
-> result mapped to `ResponseDTO`  
-> frontend displays message/data
